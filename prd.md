# 프로젝트: 기업용 제미나이(Gemini) 프롬프트 라이브러리

## 1. 프로젝트 개요
- **목표**: 기업 교육용 웹 기반 "프롬프트 갤러리"를 구축하여, 임직원들이 제미나이의 잠재력을 쉽게 확인("Wow" 모먼트)하고, 프롬프트를 즉시 복사하여 업무에 활용할 수 있도록 함.
- **참고 스타일**: [Banana X 프롬프트 패턴 컬렉션](https://furoku.github.io/bananaX/projects/infographic-evaluation/ko/)
- **핵심 가치**: "가져다 쓸 결과물을 먼저 보고, 입력(프롬프트)을 복사한다."

## 2. 타겟 사용자
- **주 사용층**: 사내 AI 교육을 수강하는 임직원.
- **부 사용층**: 업무 중 빠른 해결책이 필요한 모든 임직원.
- **주요 대상 부서**:
  - 마케팅 (Marketing)
  - 영업 (Sales)
  - 인사 (HR)
  - 개발 (Development)
  - 디자인 (Design)
  - 일반 사무 (General Business - 이메일, 보고서 작성 등)

## 3. 핵심 기능

### A. 프롬프트 갤러리 (메인 페이지)
- **비주얼 그리드 레이아웃**: 프롬프트 카드를 벽돌형(Masonry) 또는 격자형(Grid)으로 배치.
- **카드 구성**:
  - 고품질 결과 이미지 ("Wow" 요소를 보여줄 수 있는 시각적 결과물).
  - 제목/목표 (예: "콜드 메일 초안 작성하기").
  - 부서 태그 (예: "Sales").
- **필터링 및 탐색**:
  - 부서별 필터 (마케팅, 영업 등).
  - "전체 보기" 옵션.
  - 검색창 (키워드 검색).

### B. 상세 보기 (모달/오버레이)
- **인터랙션**: 현재 페이지 위로 부드럽게 열리는 모달 방식 (참고 사이트와 유사하게 맥락 유지).
- **왼쪽 영역**:
  - 결과 이미지/산출물의 확대 보기.
  - (선택 사항) 효율성, 창의성 점수 등 시각적 지표 시각화.
- **오른쪽 영역**:
  - **프롬프트 제목** 및 **상황 설명**.
  - **프롬프트 코드 블록**: 눈에 띄는 디자인 (예: 다크 모드 블록), 원클릭 "복사(Copy)" 버튼.
  - **사용 가이드/팁**: 간단한 사용법 또는 효과적인 이유 설명.
  - **부서/카테고리** 태그.

### C. 데이터 관리 (초기 단계)
- **데이터 소스**: JSON 파일을 사용하여 프롬프트 데이터를 구조화 (쉽게 수정 및 추가 가능).
- **데이터 필드**:
  - `id`: 고유 식별자.
  - `title`: 표시 제목.
  - `category`: 부서 (Enum: Marketing, HR 등).
  - `image`: 결과 이미지 경로.
  - `prompt_text`: 실제 복사할 프롬프트 내용.
  - `description`: 간단한 설명.

## 4. 사용자 흐름 (User Flow)
1. **접속 (Landing)**: 사용자가 접속하자마자 전체 프롬프트 카드 갤러리를 **즉시 확인** (별도 탐색 불필요).
2. **탐색 (Discovery)**: 필요에 따라 "마케팅" 등 필터를 눌러서 원하는 것만 골라봄 (선택 사항).
3. **선택 (Selection)**: 인상적인 광고 카피 이미지를 발견하고 카드를 **클릭**.
4. **확인 (Action)**: 모달이 열리고 -> 해당 이미지를 생성한 프롬프트를 확인.
5. **활용 (Consumption)**: "프롬프트 복사" 버튼 클릭 -> 제미나이에 붙여넣기.

## 5. 디자인 및 심미성 (비기능 요구사항)
- **테마**: 프리미엄, 깔끔함, 기업용이지만 현대적인 느낌.
- **색상**: 제미나이 브랜드와 어울리는 깊은 블루/퍼플 톤 또는 생동감 있는 포인트 컬러를 쓴 깔끔한 화이트/그레이.
- **타이포그래피**: 가독성이 좋은 모던한 산세리프(Inter/Roboto/Pretendard); 제목 등에는 세련된 느낌을 위해 세리프 사용 고려.
- **애니메이션**:
  - 카드 호버 효과 (살짝 떠오르거나 은은한 광채).
  - 필터링 시 부드러운 레이아웃 전환.
  - 모달 등장 시 페이드/슬라이드 효과.

## 6. 기술 스택 제안
- **프레임워크**: React (Vite) - 빠른 성능과 풍부한 인터랙션 구현.
- **스타일링**: Vanilla CSS (CSS Variables) - 프리미엄 디자인을 위한 정밀한 커스텀 제어.
- **데이터**: 로컬 JSON (Client-side) - 별도 백엔드 없이 빠르고 간편하게 배포 가능.
- **배포**: Vercel / Netlify / GitHub Pages.
