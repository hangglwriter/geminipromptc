# 🚀 코딩 초보자를 위한 안티그래비티(Antigravity) 200% 활용법

안티그래비티는 단순한 자동완성 도구가 아니라, 여러분과 함께 일하는 **숙련된 개발자 파트너**입니다. 코딩 지식이 없어도 이 도구를 잘 쓰면 훌륭한 결과물을 만들 수 있습니다. 초보자가 알아두면 좋은 소통의 기술을 정리해 드립니다.

---

## 1. "어떻게(How)"보다 "무엇을(What)" 말하기
코드를 어떻게 짜야 할지 고민하지 마세요. 그건 제가 합니다. 여러분은 **만들고 싶은 결과물**을 명확하게 설명해 주세요.

- **❌ 나쁜 예**: "h1 태그 폰트 사이즈를 24px로 하고 마진을 10px 줘." (전문 용어를 억지로 쓰면 오해가 생길 수 있어요)
- **✅ 좋은 예**: "제목이 너무 작아서 눈에 안 띄어. 좀 더 크고 시원하게 만들어줘."
- **✅ 좋은 예**: "버튼을 눌렀을 때 깜빡이는 효과를 줘서 눌렸다는 느낌을 확실히 주고 싶어."

## 2. 작게 쪼개서 시키기 (Iterative Development)
한 번에 거대한 사이트를 만들어달라고 하면 저도 실수를 할 수 있습니다. 레고 블록을 쌓듯이 하나씩 요청하세요.

1.  "일단 화면 중앙에 큰 '시작하기' 버튼 하나만 만들어줘."
2.  (확인 후) "좋아, 이제 그 버튼 색깔을 파란색으로 바꾸고, 누르면 입력창이 뜨게 해줘."
3.  (확인 후) "입력창 디자인을 좀 더 둥글게 다듬어줘."

## 3. 모르면 "설명해줘"라고 묻기
제가 쓴 전문 용어나 코드가 이해되지 않으면 언제든 물어보세요. 저는 24시간 대기 중인 1:1 과외 선생님이기도 합니다.

- "방금 수정한 `useEffect` 코드가 무슨 역할을 하는 거야?"
- "이 기능을 구현하는 다른 더 쉬운 방법은 없어?"
- "배포(Deployment)가 정확히 무슨 뜻이야? 초보자 눈높이로 설명해줘."

## 4. 계획(Plan)을 먼저 확인하기
제가 복잡한 작업을 시작하기 전에 보통 **"구현 계획(Implementation Plan)"**을 먼저 보여드립니다. 이때 꼼꼼히 체크하세요.

- 제가 여러분의 의도를 제대로 파악했는지 확인하는 단계입니다.
- "내 생각은 A인데 너는 B를 하려고 하네? 다시 계획해줘."라고 수정할 수 있는 가장 좋은 타이밍입니다.

## 5. 에러가 나면 "그대로 복사"해주기
- 무언가 작동하지 않을 때, 에러 메시지나 화면에 뜬 빨간 글씨를 그대로 복사해서 보여주세요.
- 혹은 "버튼을 눌렀는데 아무 반응이 없어"처럼 **증상**을 구체적으로 묘사해 주시면 해결이 빠릅니다.

---

## 💡 요약
1. **결과물 중심**으로 말하세요. (Display)
2. **하나씩 차근차근** 시키세요. (Step-by-step)
3. 모르면 **무조건 물어보세요**. (Ask)
4. 제가 제안한 **계획을 검토**하세요. (Review)

이 4가지만 기억하면, 코딩을 몰라도 원하는 모든 것을 만드실 수 있습니다!

---

## 6. 고급 기능 입문 (심화편)

질문하신 **멘션(@)**과 **워크플로우**는 작업을 더 스마트하게 만드는 도구입니다.

### 📌 멘션 (@Mention)
대화창에 `@`를 입력하면 특정 파일이나 폴더를 콕 집어서 지정할 수 있습니다.
- **언제 쓰나요?**: "이 파일 좀 봐줘"라고 말하고 싶을 때.
- **비유**: 대화하다가 손가락으로 특정 서류를 가리키며 "이거 말이야"라고 하는 것과 같습니다.
- **예시**:
    - "이 에러 왜 나는 거야?" (X - 온 동네를 다 뒤져야 함)
    - "`@App.jsx`에서 에러가 나는데 왜 이래?" (O - 바로 그 파일만 분석해서 빠름)

### ⚡ 워크플로우 (Workflow)
자주 하는 작업의 순서를 미리 적어둔 **'작업 지시서'** 혹은 **'매크로'**입니다.
- **언제 쓰나요?**: 복잡한 과정을 매번 설명하기 귀찮을 때, 버튼 하나로 해결하고 싶을 때.
- **비유**: 신입 사원에게 매번 말로 설명하는 대신 "이 매뉴얼 3페이지 대로 해"라고 하는 것.
- **예시**:
    - "배포해줘" (매번 배포 과정을 설명하거나 AI가 고민해야 함)
    - `/deploy` 워크플로우 실행 (미리 정해진 '빌드 -> 테스트 -> 업로드' 과정을 기계적으로 수행)
    - **우리가 한 것**: 아까 만든 `prompts.csv`를 JSON으로 바꾸는 과정도 워크플로우로 만들 수 있습니다!
